| 文件 | 说明 | 物理依赖 |
|------|------|----------|
| `roboclean_bt/bt_server.py` | 小车端蓝牙 SPP 服务 | 树莓派板载蓝牙、蓝牙名称 |
| `BleProtocol.kt` | App 端协议定义 | 帧格式与小车端对齐 |
| `BluetoothService.kt` | App 端蓝牙连接管理 | UUID、Android 蓝牙权限 |
| `MainActivity.kt` | 仪表盘接入真实数据 | BluetoothService 生命周期 |
| `DashboardScreen.kt` | 显示 RobotStatus | — |

**协议**：
- 帧: `[0xAA][Len][Cmd][Payload][XOR]`
- App→车: 0x01=查询 0x02=设时间 0x03=设路线 0x04=急停 0x05=启停
- 车→App: 0x11=状态(battery%+V+km+working+temp) 0x12=ACK
- 每 2 秒自动查询状态

## 🔄 驱动器手册分析 + 48V 系统修订 (2026-06-17 深夜)

用户提供了驱动器说明书（`C20-800LRC.docx`），已完成全文提取和分析。

### 说明书关键发现

| 参数 | 实际值 |
|------|--------|
| 型号 | ZBLD.C20-800LRC |
| 电压 | DC 24V~48V → **48V 系统确认** |
| 48V 功率 | **800W**（24V 仅 400W） |
| CANopen 使能 | 功能码 **F08.13=1** |
| PDO 数量 | 4 RPDO + 4 TPDO |
| COB-ID | RPDO1=200h+ID, TPDO1=180h+ID |
| 电池监测 | 寄存器 **2204h** = 母线电压（0.1V 精度） |
| 状态字 | 2100h（正转/反转/故障）, 2101h（过载/控制源） |

### 48V 系统更新

- 电池确认：48V（可能 4×12V 串联）
- 48V 下驱动器输出 800W，500W 电机余量充足
- DC-DC：48V→5V 代替 24V→5V
- 充电器：48V 铅酸充电器
- 接触器：48V 线圈代替 24V

## ⚠️ 物理布局依赖清单 (装车前必读)

> 以下所有数值都是**根据经验预估的**，必须在硬件装车后用卷尺实测，然后修改对应的参数文件。

### 📐 需要你测量的项目

| # | 测量什么 | 当前估算值 | 在哪个文件 | 怎么测 |
|---|---------|-----------|-----------|--------|
| 1 | **轮胎半径** | 0.19m | `robot_params.yaml` → `wheel_radius` | 用卷尺从轮心量到轮胎外沿，300-8 轮胎实测 |
| 2 | **左右轮距** | 0.65m | `robot_params.yaml` → `wheel_separation` | 量左右驱动轮中心之间的距离 |
| 3 | **减速比** | 56.0 | `robot_params.yaml` → `gear_ratio` | 看电机铭牌，中大力德电机上会标 |
| 4 | **编码器分辨率** | 131072 | `encoder_odom.py` 参数 | 查编码器手册，17-bit = 131072 pulses/圈 |
| 5 | **围栏在哪一侧** | left | `fence_follower.py` → `fence_side` | 站在小车后面看，围栏在左手边就是 left |
| 6 | **离围栏目标距离** | 0.50m | `fence_follower.py` → `target_distance` | 推料滚筒伸到围栏边，量小车侧面到围栏的距离 |
| 7 | **LiDAR 安装位置** | — | `slam_params.yaml` | 量雷达在小车上的 X/Y/Z 坐标（相对驱动轮轴中心） |
| 8 | **LiDAR 朝向** | 前方为 0° | `fence_follower.py` 角度筛选 | 雷达数据线的朝向 = 小车前方 |
| 9 | **超声波×4 安装位置** | — | `safety_sensor.py` GPIO 引脚 | 前/后/左/右 四个位置，左=围栏侧 |
| 10 | **安全触边安装** | 前左+前右 | `safety_sensor.py` PIN 常量 | 触边在前保险杠左右各一条 |
| 11 | **急停按钮位置** | — | `safety_sensor.py` `PIN_EMG_BUTTON` | 装在小车顶部醒目的地方，方便按 |
| 12 | **充电桩坐标** | (0,0) | `charging_dock.py` → `dock_x/y` | 建好地图后，在地图上标记充电桩的实际位置 |
| 13 | **推料滚筒转速** | 800 RPM | `fence_follower.py` → `drum_rpm` | 装车后看饲料推动效果调整，太快会把料甩飞 |
| 14 | **小车半径** | 0.45m | `nav2_params.yaml` → `robot_radius` | 量小车几何中心到最远角的距离 |

### 🔧 怎么改这些值

所有参数都在一个文件 → **`src/robot/src/roboclean_bringup/config/robot_params.yaml`**

```
chassis:
  wheel_radius: 0.19         ← 改这里
  wheel_separation: 0.65     ← 改这里
  gear_ratio: 56.0           ← 改这里

fence_following:
  fence_side: "left"         ← 改这里
  target_distance: 0.50      ← 改这里
```

改了 YAML 文件后，**重启 ROS 2 节点就生效**，不需要重新编译。

### 🚨 不改会怎样

| 参数偏差 | 后果 |
|---------|------|
| 轮胎半径差 2cm | 里程计每走 1m 就偏差 10cm，SLAM 地图会扭曲 |
| 轮距差 5cm | 转弯时角度误差累积，围栏跟随时越走越歪 |
| 围栏距离设错 | 要么撞围栏，要么离太远推不到料 |
| LiDAR 坐标不准 | 障碍物位置偏移，可能撞墙 |
| 超声波装错面 | 围栏跟随时会"看错方向" |

> 💡 **建议**：装车后先推着小车在直线地板上走 5 米，看里程计显示的距离对不对。不对就微调 `wheel_radius`，调到误差 < 2% 为止。
