# 📝 产品需求规范

> 本文档定义智能清洁小车的完整功能需求，是开发和验收的唯一依据。

---

## 1. 产品概述

| 属性 | 描述 |
|------|------|
| **产品名称** | 智能清洁小车（暂定） |
| **产品类型** | 室内+室外混合环境自主清洁机器人 |
| **目标用户** | 需要自动化地面清洁的家庭/园区用户 |
| **运行环境** | 室内平面 + 室外硬化路面 |

---

## 2. 功能需求

### 2.1 安卓 App 端

#### F1 — 蓝牙连接管理

| 属性 | 描述 |
|------|------|
| **优先级** | P0（核心） |
| **描述** | 用户可以通过蓝牙搜索、连接、断开小车 |
| **详情** | 显示附近蓝牙设备列表，点击连接，状态指示（已连接/断开/连接中） |

#### F2 — 仪表盘主页

| 属性 | 描述 |
|------|------|
| **优先级** | P0（核心） |
| **描述** | 展示小车核心状态信息 |
| **包含** | 电量百分比、累计行驶里程、当前连接状态、当前工作状态 |

#### F3 — 工作时间设置

| 属性 | 描述 |
|------|------|
| **优先级** | P1（重要） |
| **描述** | 用户可设定每天小车自动工作的时间段 |
| **详情** | 支持多个时间段（如 08:00-09:00, 17:00-18:00），可启用/禁用 |

#### F4 — 路线设置

| 属性 | 描述 |
|------|------|
| **优先级** | P1（重要） |
| **描述** | 用户在地图上设置途经关键点，小车依次经过 |
| **详情** | 地图选点 → 加入途经点列表 → 可排序/删除 → 发送到小车 |

#### F5 — 低电量提醒

| 属性 | 描述 |
|------|------|
| **优先级** | P1（重要） |
| **描述** | 电量低于阈值时 App 推送通知，并显示"正在返回充电"状态 |

### 2.2 小车端

#### F6 — 自主行驶

| 属性 | 描述 |
|------|------|
| **优先级** | P0（核心） |
| **描述** | 小车根据设定的路线途经点自主规划路径并行驶 |
| **依赖** | 2D 激光雷达 + SLAM 导航 |

#### F7 — 清洁工作

| 属性 | 描述 |
|------|------|
| **优先级** | P0（核心） |
| **描述** | 行驶过程中旋转清洁刷自动运转 |
| **详情** | 清洁电机在行驶时自动启动，停止时关闭 |

#### F8 — 自动回充

| 属性 | 描述 |
|------|------|
| **优先级** | P0（核心） |
| **描述** | 电量低于阈值（如 15%）时，自动中断当前任务，返回充电桩 |
| **详情** | 通过激光雷达识别充电桩反光标记，自动泊入充电 |

#### F9 — 安全防撞

| 属性 | 描述 |
|------|------|
| **优先级** | P0（核心） |
| **描述** | 多层级安全保护 |
| **详情** | 第一层：雷达避障（软件级）；第二层：防撞条（硬件级） |

---

## 3. 非功能需求

| 类别 | 要求 |
|------|------|
| **通信距离** | 蓝牙 ≥ 10 米（空旷） |
| **续航时间** | ≥ 2 小时连续工作 |
| **充电方式** | 自动回充 + 手动充电 |
| **App 响应** | 操作响应 ≤ 1 秒 |
| **安全** | 防撞条触发后立即停车 |
| **UI** | 蓝白配色，简洁直观 |

---

## 4. 约束与假设

- **假设**：用户具备基本的手机操作能力
- **假设**：工作区域有可识别的特征（墙壁、障碍物）供雷达建图
- **约束**：蓝牙通信距离有限，超出距离后 App 无法实时查看状态
- **约束**：室外环境受天气影响（雨雪天气不建议使用）
