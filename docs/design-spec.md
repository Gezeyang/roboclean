# 🎨 UI/UX 设计规范

> 本文档定义安卓 App 的视觉设计标准、页面结构和交互规范。

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **简洁** | 减少认知负担，一屏内展示核心信息 |
| **直观** | 图标 + 文字结合，功能一目了然 |
| **大触控区** | 按钮/列表项 ≥ 48dp，方便手指点击 |
| **容错** | 危险操作（如急停）需二次确认 |

## 5. 交互规范

| 场景 | 行为 |
|------|------|
| 点击"急停" | 弹出确认对话框 → 确认后发送急停指令 |
| 低电量通知 | App 顶部弹出 SnackBar（橙色背景） |
| 蓝牙断开 | 仪表盘状态变为红色"已断开"，弹出提示 |
| 路线保存 | 保存到本地 + 自动发送到小车（如果已连接） |
| 数据加载 | 显示蓝色骨架屏/加载动画 |

## 7. Skills 约束

> 本节定义项目的 Copilot Skills 结构，AI 助手通过 SKILL.md 自动加载对应模块的约束。
> **修改规则：改这里 → 同步改 `.github/skills/<name>/SKILL.md`。**

### 7.1 Skill 划分

| Skill | 触发词 | 代码范围 | 核心约束 |
|-------|--------|----------|----------|
| `frontend` | Kotlin, Compose, Android, UI | `src/app/` | 见 §7.2 |
| `backend` | Python, ROS 2, 树莓派, 驱动 | `src/robot/` | 见 §7.3 |
| `docs` | 文档, 设计规范, devlog | `docs/` `devlog/` | 见 §7.4 |

### 7.2 Frontend 约束

- **语言**：Kotlin，禁止 Java
- **UI 框架**：Jetpack Compose + Material 3，禁止 XML layout
- **配色**：严格使用 `Color.kt` 中的 Blue 系列 + Success/Warning/Error
- **数据持久化**：DataStore Preferences，禁止 Room（项目规模不需要）
- **蓝牙协议**：帧格式不可改（`BleProtocol.kt`），与 `docs/tech-spec.md` §3 严格对齐
- **组件复用**：通用组件（`InfoCard`、`StatusCard`）不得内联重写
- **权限**：蓝牙 + 定位权限运行时申请，targetSdk=34

### 7.3 Backend 约束

- **语言**：Python 3.10+，禁止 C++（除非性能必需）
- **框架**：ROS 2 Humble，禁止 ROS 1
- **CANopen 协议**：电机驱动严格遵循 `docs/tech-spec.md` §4 和 `docs/driver-c20-800lrc.md`
- **Topic 命名**：遵循 ROS 2 规范（snake_case），前缀按模块（`/motor/`、`/sensor/`、`/nav/`）
- **蓝牙服务**：与 App 端 `BleProtocol` 帧格式保持一致
- **安全传感器**：急停、触边、超声波独立节点，互不阻塞
- **日志**：使用 `rclpy` logger，禁止 `print()`

### 7.4 Docs 约束

- **格式**：Markdown，UTF-8 编码
- **devlog**：每天一个文件 `YYYY-MM-DD.md`，模板：今日完成 / 待办 / 问题
- **术语统一**：App 端 → "App"，小车端 → "小车/robot"，蓝牙帧 → "BleProtocol"
- **禁止**：文档中写死 IP 地址、绝对路径、API Key 等敏感信息
