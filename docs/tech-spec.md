# 🔧 技术选型与架构规范

> 本文档定义项目的技术栈、系统架构、通信协议和开发规范。

---

## 1. 技术栈总览

```
┌─────────────────────────────────────────────────┐
│                  安卓 App                        │
│           Kotlin + Jetpack Compose               │
│          蓝牙 SPP / BLE 通信                      │
└────────────────────┬────────────────────────────┘
                     │ 蓝牙
┌────────────────────┴────────────────────────────┐
│               小车端（树莓派）                     │
│     OS: Ubuntu Server + ROS 2 Humble             │
│     SLAM: slam_toolbox / Cartographer            │
│     导航: Nav2                                   │
│     语言: Python 3 / C++                         │
└─────────────────────────────────────────────────┘
```

| 层面 | 选型 | 理由 |
|------|------|------|
| **App 语言** | Kotlin | Android 官方首选，现代化、安全 |
| **App UI** | Jetpack Compose | 声明式 UI，开发效率高 |
| **小车 OS** | Ubuntu Server 22.04 | 树莓派兼容好，ROS 2 支持佳 |
| **机器人框架** | ROS 2 Humble | 行业标准，SLAM/导航生态成熟 |
| **SLAM** | slam_toolbox | 2D 雷达 SLAM，ROS 2 原生支持 |
| **导航** | Nav2 (Navigation 2) | ROS 2 标准导航框架 |
| **App↔车通信** | 蓝牙 SPP（串口仿真） | 简单可靠，适合指令传输 |
| **电机控制** | CANopen (CAN Bus) | 工业标准，中大力德驱动器原生支持 |
| **供电系统** | 24V 铅酸 60Ah | 用户提供，驱动 500W×2 + 200W 电机 |
| **编码器** | 角位移绝对值编码器 | 断电不失位置，精度高 |

## 3. 蓝牙通信协议

### 3.1 数据帧格式

```
┌──────┬──────┬──────┬──────────┬──────┐
│ 帧头  │ 长度  │ 指令  │   数据    │ 校验  │
│ 0xAA │ 1Byte│ 1Byte│  N Byte  │1Byte │
└──────┴──────┴──────┴──────────┴──────┘
```

### 3.2 指令定义（草案）

| 指令 | 方向 | 说明 |
|------|------|------|
| `0x01` | App→车 | 查询状态（电量/里程/工作状态） |
| `0x02` | 车→App | 返回状态数据 |
| `0x03` | App→车 | 下发工作时间表 |
| `0x04` | App→车 | 下发路线途经点 |
| `0x05` | App→车 | 紧急停止 |
| `0x06` | 车→App | 低电量告警 |
| `0x07` | 车→App | 任务完成通知 |

## 4. CANopen 电机控制协议

### 4.1 CAN 总线拓扑

```
树莓派 (CANopen Master)
  │ CAN HAT (MCP2515)
  │
  ╧══════════════════ CAN Bus (双绞屏蔽线, 500kbps) ══════════════════
  │                    │                    │
  ├─ 终端电阻 120Ω     │                    ├─ 终端电阻 120Ω
  │                    │                    │
                 ┌─────┴─────┐        ┌─────┴─────┐        ┌─────┴─────┐
                 │  驱动器1   │        │  驱动器2   │        │ 清洁驱动  │
                 │ Node ID: 1 │        │ Node ID: 2 │        │ Node ID: 3 │
                 │ 左轮 500W  │        │ 右轮 500W  │        │ 刷子 200W  │
                 └───────────┘        └───────────┘        └───────────┘
```

### 4.2 CANopen 通信机制

| 机制 | 说明 |
|------|------|
| **NMT** | 网络管理：控制驱动器上/下线、复位 |
| **SDO** | 服务数据对象：配置驱动器参数（加速、减速、PID） |
| **PDO** | 过程数据对象：实时传输速度指令 / 位置反馈（高优先级） |
| **Heartbeat** | 心跳：驱动器定时发送状态，监控在线 |

### 4.3 期望的 PDO 映射（需与驱动器手册核对）

| PDO | 方向 | 内容 |
|-----|------|------|
| TPDO1 | 驱动器→主机 | 当前位置（绝对值编码器）+ 当前速度 + 状态字 |
| RPDO1 | 主机→驱动器 | 目标速度 + 控制字（使能/启停/模式） |

### 4.4 驱动器关键寄存器（ZBLD.C20-800LRC）

> 详见 [`docs/driver-c20-800lrc.md`](driver-c20-800lrc.md)

| 地址 | 含义 | R/W | 说明 |
|------|------|-----|------|
| `2000h` | 控制命令 | W/R | 0001h=正转, 0002h=反转, 0005h=停机, 0006h=紧停 |
| `2001h` | 设定速度 | W/R | 单位 RPM |
| `2004h` | 实际速度 | R | 单位 RPM |
| `2005h` | 实际电流 | R | 单位 0.1A |
| `2006h` | 母线电压 | R | 单位 0.1V（可用于监测电池） |
| `2007h` | 故障代码 | R | 0=正常 |
| `200Bh` | 状态字 | R | Bit0=运行, Bit1=正转, Bit3=故障 |

### 4.5 软件实现

```
树莓派端：
  can0 接口 (SocketCAN) → CANopen 库 (canopen 或 lely-core)
       ↓
  ROS 2 节点: canopen_master_node
       ↓
  发布/订阅:
    /motor/left/odom      ← 左轮里程（从编码器计算）
    /motor/right/odom     ← 右轮里程
    /motor/cmd_vel        → 速度指令
    /motor/status         ← 驱动器状态
```

## 5. 供电系统

### 5.1 供电架构

| 电压 | 来源 | 供给 |
|------|------|------|
| 24V DC | 铅酸电池 60Ah | 三路驱动器（驱动轮×2 + 清洁×1）、接触器线圈 |
| 5V DC | 24V→5V DC-DC 隔离模块 | 树莓派 4B、镭神 N10P |
| 3.3V | 树莓派板载 | CAN HAT、GPIO 传感器 |

### 5.2 安全回路

```
电池(+) → 总保险 100A → 急停按钮(常闭) → 接触器线圈 → 电池(-)
                                                  │
                                  接触器主触点(三路) → 各驱动器电源
```

- 按下急停 → 接触器断开 → 所有驱动器断电
- 安全触边触发 → GPIO → 软件急停 + 接触器断开

## 6. 开发规范

### 6.1 代码规范

| 项目 | 规范 |
|------|------|
| App 代码风格 | Kotlin 官方代码规范 |

| 类型 | 命名规则 | 示例 |
|------|----------|------|

| 小车节点 | `xxx_node.py` | `motor_control_node.py` |
| 小车启动 | `xxx.launch.py` | `bringup.launch.py` |
